# Test
# Исследование влияния децимации и интерполяции на точность восстановления сигнала

## 📋 Описание проекта

Данный проект демонстрирует эффекты, возникающие при последовательном применении децимации (понижения частоты дискретизации) и интерполяции (повышения частоты дискретизации) к гармоническому сигналу. Проект реализован на языке MATLAB и включает анализ ошибки восстановления в зависимости от частоты входного сигнала.

## 🎯 Цель работы

Исследовать влияние коэффициента децимации/интерполяции (3 раза) на точность восстановления синусоидального сигнала и теоретически обосновать полученные зависимости ошибки от частоты.

## 📊 Задачи проекта

1. **Генерация тестового сигнала**
   - Создание синусоидального сигнала частотой 0.5 Гц
   - Частота дискретизации: 1 кГц
   - Длительность сигнала: 10 секунд

2. **Реализация децимации**
   - Понижение частоты дискретизации в 3 раза
   - Использование антиалиасного фильтра нижних частот

3. **Реализация интерполяции**
   - Повышение частоты дискретизации в 3 раза
   - Использование сглаживающего фильтра нижних частот

4. **Анализ ошибок**
   - Измерение среднеквадратичной ошибки (RMSE) между исходным и восстановленным сигналом
   - Исследование зависимости ошибки от частоты входного сигнала (0.1 - 100 Гц)

## 🔧 Используемые методы

### Децимация (Downsampling)
- **Алгоритм**: Фильтрация ФНЧ + прореживание отсчётов
- **Фильтр**: Баттерворта 8-го порядка
- **Частота среза**: `Fs_orig / (2 * factor) = 166.67 Гц`
- **Назначение**: Подавление высокочастотных компонент перед прореживанием для предотвращения эффекта наложения спектров (aliasing)

### Интерполяция (Upsampling)
- **Алгоритм**: Вставка нулей + фильтрация ФНЧ
- **Коэффициент усиления**: Умножение на `factor` для компенсации энергии
- **Частота среза**: `Fs_orig / 2`
- **Назначение**: Сглаживание "ступенек" после вставки нулевых отсчётов

## 📈 Результаты

### Сравнение сигналов (f = 0.5 Гц)
- **Визуальное сравнение**: Исходный и восстановленный сигналы практически идентичны
- **Амплитудные искажения**: Отсутствуют на низких частотах
- **Фазовые искажения**: Минимальны благодаря использованию `filtfilt` (нулевая фаза)

### Зависимость ошибки от частоты

| Диапазон частот | Характер ошибки | Объяснение |
|----------------|----------------|------------|
| 0.1 - 50 Гц | Очень малая (< 0.01) | Сигнал полностью попадает в полосу пропускания фильтров |
| 50 - 166 Гц | Умеренный рост | Начало спада АЧХ фильтра дециматора |
| 166 - 200 Гц | Резкий рост на 2-3 порядка | Частота приближается к частоте среза фильтра |
| > 200 Гц | Ошибка ≈ амплитуде сигнала | Сигнал полностью подавлен фильтром |

## 🧠 Теоретическое объяснение

### 1. Теорема Котельникова (Найквиста)
- **Исходная частота дискретизации**: 1000 Гц → граница Найквиста: 500 Гц
- **После децимации (↓3)**: частота дискретизации = 333.33 Гц → граница Найквиста: **166.67 Гц**

### 2. Эффект наложения спектров (Aliasing)
Частоты выше 166.67 Гц после децимации "заворачиваются" в диапазон 0-166.67 Гц, создавая необратимые искажения. Антиалиасный фильтр подавляет эти частоты, но вместе с ними ослабляет и полезные компоненты сигнала в переходной зоне.

### 3. Почему ошибка растёт именно около 166.67 Гц?

При приближении частоты сигнала к 166.67 Гц:
- Амплитуда сигнала уменьшается из-за спада АЧХ
- Возникают фазовые искажения
- После интерполяции сигнал не может быть полностью восстановлен

### 4. Неидеальность реальных фильтров
- Фильтр Баттерворта 8-го порядка имеет **конечную крутизну спада** (≈ 48 дБ/октаву)
- Полоса перехода от пропускания к подавлению не мгновенна
- Возникают остаточные пульсации из-за нелинейности ФЧХ (несмотря на `filtfilt`)

## 💻 Используемые технологии

- **Язык программирования**: MATLAB R2020a и новее
- **Основные функции**:
  - `butter` - расчёт коэффициентов фильтра Баттерворта
  - `filtfilt` - фильтрация с нулевым фазовым сдвигом
  - `decimate` (альтернатива) - встроенная функция децимации
  - `interp` (альтернатива) - встроенная функция интерполяции



## 📚 Заключение

Проект наглядно демонстрирует ключевой принцип цифровой обработки сигналов: **перед децимацией обязательна фильтрация для подавления частот выше новой границы Найквиста**. Невыполнение этого требования приводит к необратимым искажениям (алиасингу), а даже при правильной фильтрации сигналы в переходной зоне АЧХ восстанавливаются с ошибкой.

Полученные результаты подтверждают теорию: ошибка минимальна для частот, не превышающих `Fs_orig / (2 * factor)`, и резко возрастает за этой границей.
